# BUG-2026-07-05-001 — нижняя панель шире экрана приложения

## Цель

Исправить ширину нижней панели: она должна быть по ширине экрана приложения, а не по ширине всего окна браузера.

## Контекст

Алексей прислал скрин 2026-07-05. На desktop/web сам экран приложения отображается как узкий мобильный контейнер по центру, но нижняя панель растянута почти на всю ширину браузера.

Это нужно исправить на всех экранах, где видна нижняя навигация.

## Приоритет

- Severity: Medium
- Priority: P2
- Area: UI / Navigation / Responsive shell
- Backlog: BACK-046
- Владелец: Юрий

## Где смотреть

- `FILE_MAP_UI.md` — сначала открыть карту UI.
- `index.html`:
  - `bottom-nav-v2` внутри `#home`;
  - `global-nav` для остальных экранов;
  - `showScreen()` и скрытие nav на экранах без нижней панели.
- `styles.css` / `styles/main.less` / `styles/layout.less` / `styles/screens/voice.less` — проверить, какой CSS реально задаёт позиционирование нижней панели в текущей сборке.

Важно: в проекте уже есть два nav-компонента. При правке меню или его контейнера нужно проверять оба: `bottom-nav-v2` и `global-nav`.

## Ожидаемое поведение

- Desktop/web: нижняя панель не выходит за границы центрального app-контейнера.
- Home: `bottom-nav-v2` совпадает по ширине с экраном приложения.
- Остальные экраны с nav: `global-nav` совпадает по ширине с экраном приложения.
- Mobile Telegram WebView: нижняя панель остаётся по ширине мобильного экрана.
- Safe-area снизу не ломается.
- В AI-чате при фокусе в поле ввода прежнее скрытие нижней панели не регрессирует.

## Что не делать

- Не растягивать сам экран приложения ради панели.
- Не чинить только один экран.
- Не трогать логику навигации, если проблема решается CSS-контейнером.

## Acceptance criteria

1. Desktop/web 1440px+: нижняя панель по ширине равна app-контейнеру и центрирована вместе с ним.
2. Desktop/web 1024px: панель не выходит за границы app-контейнера.
3. Mobile 390px: панель занимает доступную ширину экрана и не обрезается.
4. Проверены Home, Calendar, AI-chat, task-detail/profile или любой другой экран, где виден `global-nav`.
5. `git diff --check` проходит.
6. Если менялся `index.html`, выполнена проверка кириллицы до/после по правилу проекта.
