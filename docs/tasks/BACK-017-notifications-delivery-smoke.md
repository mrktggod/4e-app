# BACK-017 — notification delivery smoke

Цель: довести `BACK-017` до честного QA-сценария без догадок по исходникам. API settings уже проверялись, но `Done` возможен только после реальной доставки Telegram-уведомления на привязанный аккаунт.

## 1. Что уже считается готовым

| Layer | Evidence |
| --- | --- |
| Settings API | `/notifications/settings` GET/PUT сохраняет настройки в D1 |
| D1 migrations | Актуальны для settings path |
| Briefing check endpoint | `/briefings/check` живой |
| UI intent | Настройки уведомлений можно включать/изменять в продукте |

## 2. Что нельзя закрыть без ручного smoke

| Gap | Why manual |
| --- | --- |
| Telegram delivery | Нужно подтвердить реальное сообщение в Telegram на привязанном пользователе |
| Correct recipient | Нужно убедиться, что сообщение ушло не в тестовый/старый chat id |
| User-facing copy | Нужно увидеть финальный текст в Telegram client |
| Timing | Нужно проверить, что scheduled/check path реально приводит к доставке |

## 3. Минимальный smoke утром

| Step | Action | Expected |
| --- | --- | --- |
| 1 | Войти staging-пользователем с привязанным Telegram | Пользователь распознан, chat id доступен backend-у |
| 2 | Открыть настройки уведомлений | Текущие настройки загружаются без ошибки |
| 3 | Включить утренний/тестовый briefing | PUT сохраняет значение, UI не откатывается |
| 4 | Запустить безопасный check path или дождаться ближайшего окна | Backend пытается доставить уведомление |
| 5 | Проверить Telegram | Сообщение пришло правильному пользователю |
| 6 | Отключить настройку обратно, если это личный аккаунт | После smoke не остается нежелательных уведомлений |

## 4. Pass/fail запись

| Result | Backlog action |
| --- | --- |
| Delivery пришла правильному пользователю | `BACK-017` можно перевести из `QA partial` в `Done` |
| Settings сохраняются, но delivery нет | Завести P1 bug в `pm/bugs.md`, `BACK-017` остается `QA partial` |
| Delivery пришла не туда | P0/P1 privacy bug, beta invite стоп |
| Copy непонятная, но доставка работает | P2 polish bug, не блокирует core beta |

## 5. Safety notes

- Не использовать production-пользователей для первого smoke, если staging достаточно.
- Не логировать `BOT_TOKEN` или chat id в публичные файлы.
- Не запускать массовые scheduled checks на всех пользователей ради одного smoke.
- После проверки вернуть личные настройки уведомлений в нормальное состояние.
