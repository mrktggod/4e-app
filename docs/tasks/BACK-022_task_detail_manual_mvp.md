# BACK-022 — Ручной MVP детальной карточки задачи

## Задача

Сделать экран `task-detail` полноценным для ручного использования, чтобы пользователь мог вести задачу без голосового ввода.

## Контекст

`BACK-019` улучшает карточки задач в списке: номер, приоритет, тег, дедлайн, просрочка и swipe-действия. `BACK-022` не дублирует `BACK-019`: это задача про детальный экран задачи, который пользователь открывает после нажатия на карточку.

Голос не всегда доступен или удобен. Если пользователь работает руками, 4 не должен проигрывать базовым напоминаниям и заметкам по ключевому функционалу.

## Объём первой волны

- Редактирование полей без browser `prompt`.
- Быстрые сроки: сегодня, завтра, на неделе, выбрать дату, без срока.
- Время и напоминание: в момент срока, за 15 минут, за час, за день, своё время, без напоминания.
- Чек-лист внутри задачи.
- Приоритет: обычная, важная, горит.
- Статусы: активная, в ожидании, сделана, отложена, отменена.
- Русские направления вместо технических `incoming` / `outgoing`.
- Описание задачи остаётся ручным текстовым полем.
- AI-действие: разбить задачу на шаги.

## Что не входит в первую волну

- Повторы задач.
- Вложения и файлы.
- Умные списки.
- Поиск и фильтры по тегам.
- Календарная синхронизация.
- Командные комментарии и совместная работа.

Эти функции вынесены в `BACK-023`.

## Критерий готовности

Пользователь открывает детальную карточку задачи и за 20-40 секунд руками настраивает срок, время, напоминание, статус, приоритет, направление, чек-лист и описание. Изменения сохраняются, корректно отображаются после перезагрузки и не ломают создание задач через голос/AI.

## PM acceptance checklist

1. Поля редактируются без `prompt`.
2. Есть быстрый выбор срока и ручной выбор даты.
3. Время и напоминание выбираются отдельно.
4. Чек-лист можно добавить, отметить и сохранить.
5. Приоритет и статус видны в карточке.
6. Направления показываются на русском языке.
7. AI-разбивку на шаги можно принять или не использовать.
8. Старая задача без новых полей открывается без пустого экрана.
9. После reload данные остаются на месте.
