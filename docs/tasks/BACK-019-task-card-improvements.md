## Старт сессии

```bash
git fetch origin && git switch main && git pull --rebase
git switch -c feat/task-card-improvements
```

## Что нужно сделать

### 1. Новая компоновка карточки

```
Строка 1: [Номер с цветом приоритета] [Тег-категория]    [Дата дедлайна]
Строка 2-3: Название задачи (2 строки max, обрезать с …)
```

### 2. Приоритет — цветная точка/обводка на номере

| Приоритет | Цвет |
|---|---|
| P0 / срочно | Красный `#E53E3E` |
| P1 | Оранжевый `#DD6B20` |
| P2 | Жёлтый `#D69E2E` |
| P3 / обычная | Без выделения |

Определять приоритет из поля `task.priority` (если есть) или из тега `срочно`/`горит` в тексте.

### 3. Просроченные задачи

Если дедлайн прошёл — вся карточка получает приглушённый красный фон (не яркий, фоновый оттенок). CSS-класс `task-card--overdue`.

### 4. Дедлайн — относительная дата

| Ситуация | Показывать |
|---|---|
| Сегодня | `сегодня` |
| Вчера | `вчера` |
| Завтра | `завтра` |
| Просрочено | `просрочено на N дн.` |
| Дальше чем завтра | `15 июл` |
| Нет дедлайна | ничего |

Логика уже частично есть в `index.html` (Фаза 11, коммит `7ebc4a7`) — расширить её.

### 5. Свайп влево → кнопки действий

При свайпе влево карточка сдвигается, из-под неё появляются кнопки:
- **Отменить** (серая) — ставит статус `cancelled`, убирает карточку
- **Перенести** (синяя) — открывает date picker для выбора нового дедлайна

### 6. Свайп вправо → Завершить

При свайпе вправо появляется кнопка:
- **Завершить** (зелёная) — ставит статус `done`, убирает карточку с анимацией

## Критерий готовности

- Карточки отображают приоритет цветом
- Просроченные — красный фон
- Дедлайн — относительная дата
- Свайп влево: кнопки Отменить / Перенести
- Свайп вправо: кнопка Завершить
- Кириллица цела после правки (проверка по AGENTS.md Шаг 0)
