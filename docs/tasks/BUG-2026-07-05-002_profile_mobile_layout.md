# BUG-2026-07-05-002 — мобильная верстка профиля выглядит неаккуратно

## Цель

Привести экран профиля на мобильной веб-версии к аккуратной форме: ровные отступы, понятные строки полей, статусные бейджи без наложений и гармоничная нижняя часть формы.

## Контекст

Алексей прислал два скрина 2026-07-05. Это веб-версия на мобильной ширине. На экране "Профиль" визуально заметны проблемы:

- строка Telegram и бейдж "Не привязан" выглядят съехавшими;
- заголовок "Дата рождения" слишком близко к предыдущему блоку;
- заголовок "О себе" визуально прилипает к верхней границе секции;
- textarea "О себе" и счётчик `0 / 200` не выглядят как аккуратный единый блок;
- нижняя часть формы должна проверяться вместе с кнопкой "Сохранить профиль" и нижней навигацией.

## Приоритет

- Severity: Medium
- Priority: P2
- Area: UI / Profile / Mobile layout
- Backlog: BACK-043
- Владелец: Юрий

## Где смотреть

- `FILE_MAP_UI.md` — сначала открыть карту UI.
- `index.html` — секция экрана профиля и связанные функции сохранения профиля.
- CSS профиля: проверить текущий источник стилей, который реально влияет на mobile layout (`styles.css`, `styles.min.css`, LESS-модули, если правка идёт через исходники стилей).

Важно: если меняется `index.html`, выполнить обязательную проверку кириллицы до и после правки по правилу проекта.

## Ожидаемое поведение

- Поля "Телефон", "Email" и "Telegram" имеют единый визуальный ритм.
- Статусы "Не подтверждён" / "Не привязан" не налезают на input и не выглядят случайно приклеенными.
- "Дата рождения" начинается с нормального вертикального отступа после предыдущего блока.
- "О себе" не прижат к границе секции.
- Textarea и счётчик `0 / 200` визуально принадлежат одной секции.
- Кнопка "Сохранить профиль" доступна после прокрутки и не конфликтует с нижней навигацией.

## Что не делать

- Не исправлять только один бейдж, если общий ритм формы остаётся неровным.
- Не уменьшать шрифт ради того, чтобы всё влезло любой ценой.
- Не ломать desktop/web и Telegram WebView ради mobile web.
- Не менять бизнес-логику профиля, если задача решается версткой.

## Acceptance criteria

1. Mobile web 390px: блок редактирования профиля выглядит аккуратно от аватара до кнопки сохранения.
2. Phone/email/telegram: input и статусный бейдж выровнены, не пересекаются и имеют понятные отступы.
3. "Дата рождения": заголовок не прилипает к предыдущей карточке.
4. "О себе": заголовок, textarea и счётчик символов стоят гармонично и не перекрываются нижней навигацией.
5. Desktop/web: профиль не получил регрессий в ширине, отступах и сохранении.
6. Telegram WebView: профиль остаётся читаемым на реальном мобильном экране.
7. `git diff --check` проходит.
8. Если менялся `index.html`, выполнена проверка кириллицы до/после по правилу проекта.
