# BACK-045 — Авторизация через сервисы РФ: VK ID + Яндекс ID

## Контекст

Для РФ-аудитории 4 должен давать привычные и доверенные способы входа. VK уже есть в продуктовой модели, но вход через российские провайдеры нужно оформить как отдельное направление: VK ID + Яндекс ID, без удаления Email и Telegram.

## Цель

Пользователь может войти или зарегистрироваться через VK ID и Яндекс ID, а система сохраняет единую модель аккаунта и не создаёт дубль при повторном входе через другой провайдер.

## Что сделать

1. Проверить текущую реализацию VK-входа и понять, достаточно ли она соответствует VK ID-сценарию.
2. Спроектировать вход через Яндекс ID: redirect/callback, хранение `yandexId`, обработка ошибок, privacy-текст.
3. Связать VK ID и Яндекс ID с текущей моделью пользователя: Email, Telegram, VK.
4. Учесть BACK-026: при совпадении email/telegram/vk/yandex не плодить дубль, а связывать аккаунт.
5. На login/register показать понятные варианты входа без перегруза первого экрана.

## Критерии готовности

- Вход через VK ID работает в web/VK-контуре и не ломает Telegram Mini App.
- Вход через Яндекс ID работает в web-контуре.
- Повторный вход через другой провайдер не создаёт отдельного пользователя, если аккаунт можно связать безопасно.
- Пользователь в профиле видит, какие способы входа привязаны.
- Есть smoke-проверка для web, VK Mini App и Telegram Mini App.

## Вопросы перед разработкой

- Нужно ли выводить Яндекс ID уже в первом экране входа или сначала оставить как дополнительный вариант.
- Нужно ли согласование текста privacy перед включением Яндекс ID.
