# BACK-023 — Слияние аккаунтов с переносом задач

**Приоритет:** P1  
**Ответственный:** Codex  
**Ветка:** `feat/account-merge`  
**Одна задача за сессию**

## Контекст

Сейчас при входе через Telegram создаётся новый аккаунт `tg_XXXXXXXXX@telegram.local`, даже если пользователь уже зарегистрирован через email или VK с тем же telegramId. Это создаёт дубли с разными задачами и планами.

Реальный случай: у Юрия был аккаунт `84fef140` (Telegram) с задачами и аккаунт `1efd3fc2` (email + VK) без задач. Пришлось мержить вручную через wrangler D1 + KV.

## Что нужно сделать

### 1. При входе через любой провайдер — проверить дубли

В `worker.js` в обработчике логина (Telegram, VK, email) **до создания нового пользователя** выполнить поиск существующего:

```
Telegram login → ищем user по telegramId в KV (user:tg_XXXXX) и в D1 auth_identities
VK login → ищем user по vkId
Email login → ищем user по email
```

Если нашли существующего — **не создавать нового**, а вернуть сессию существующего.

### 2. Слияние при обнаружении дубля

Если пользователь уже залогинен под одним аккаунтом и логинится через другой провайдер — проверить не существует ли уже другой аккаунт с этим провайдером:

**Алгоритм слияния:**

```
1. Определить "главный" аккаунт — тот у которого plan="paid" или более свежий trialEndsAt
2. Перенести задачи из app_task_lists вторичного в главный (объединить JSON-массивы, исключить дубли по id)
3. Скопировать auth_identities вторичного в главный (D1)
4. Обновить KV главного: добавить telegramId/vkId если их не было
5. Удалить вторичный аккаунт: из KV (user:...) и пометить deleted_at в D1 users
6. Вернуть сессию главного аккаунта
```

### 3. Поля для поиска дублей

При создании пользователя в KV сохранять обратные индексы:
- `tg_rev:USER_ID` → telegramId (уже есть)
- Добавить: `by_tg:267468814` → USER_ID (обратный поиск по telegramId)
- Добавить: `by_vk:109496030` → USER_ID (обратный поиск по vkId)

### 4. Что НЕ делать

- Не сливать автоматически без явного признака что это один человек (совпадение telegramId или vkId + email)
- Не удалять вторичный аккаунт если задачи не перенесены успешно

## Критерий готовности

- Повторный вход через TG/VK не создаёт новый аккаунт если telegramId/vkId уже есть у другого пользователя
- При слиянии задачи объединяются без потерь
- В D1 `auth_identities` обновляется для главного аккаунта
- Тест: создать двух пользователей с одинаковым telegramId, войти — должен вернуть одного
