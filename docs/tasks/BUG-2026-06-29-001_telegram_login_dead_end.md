# BUG-2026-06-29-001 — Telegram login dead end

## Цель

Убрать тупик во входе через Telegram: пользователь должен получить понятное действие, а не только toast "Открой бота и нажми Start".

## Контекст

Алексей проверил вход через Telegram 2026-06-29. На экране входа после нажатия "Войти через Telegram" появляется сообщение: "Открой бота и нажми Start — получишь ссылку для входа". В самом Telegram действие не предлагается.

Текущий код `index.html` вызывает `loginWithTelegram()`. Если нет `Telegram.WebApp.initData` или `initDataUnsafe.user`, функция показывает toast и возвращает пользователя на экран login. Бот находится в отдельном репозитории `mrktggod/4e-bot`, локально не подключён.

## Приоритет

- Severity: High
- Priority: P1
- Area: Auth / Telegram / Bot handoff
- Backlog: BACK-024

## Что проверить

1. В `4e-app`:
   - кнопка "Войти через Telegram" не должна вести в тупик;
   - если нет `Telegram.WebApp.initData`, нужен явный следующий шаг: открыть `@Denzel89bot` или показать кнопку/ссылку;
   - нужен запасной путь через email-вход.
2. В `4e-bot`:
   - `/start` отвечает пользователю;
   - бот присылает рабочую ссылку Mini App/логина;
   - Railway-деплой живой;
   - deep link или кнопка открывает правильный live URL.
3. В Worker:
   - `/auth/telegram` принимает валидный `initData`;
   - ошибка при невалидном/пустом `initData` контролируемая и понятная.

## Acceptance criteria

- Нажатие "Войти через Telegram" вне валидной Mini App-сессии показывает понятное действие, а не только toast.
- Кнопка/ссылка открывает `@Denzel89bot`.
- После `/start` бот выдаёт рабочую ссылку входа или Mini App.
- Открытие ссылки из Telegram создаёт/возвращает сессию и переводит пользователя на главный экран.
- Если Telegram/bot недоступен, пользователь видит запасной email-вход.

## Ограничения

- Не править bot-логику из `4e-app` без доступа к репозиторию `mrktggod/4e-bot`.
- Push/merge не делать автоматически. Публикация через GitHub Desktop или рабочий процесс Юрия после проверки.
