# SMART-004 — group task capture smoke

Цель: закрывать лаконичную фиксацию задач в группах только после живого Telegram/group smoke. Локальный bot runtime и staging-worker smoke полезны, но не доказывают социальный UX в настоящем групповом контексте.

## 1. Что уже доказано

| Layer | Evidence |
| --- | --- |
| Local bot runtime | Стартует на staging-секретах |
| Staging worker bot action | Smoke прошёл: однострочное подтверждение, сохранение, `✏️/✕` через `delete-task` |
| Core behavior | Бот умеет лаконично сохранить задачу из группового контекста |

## 2. Live group smoke checklist

| ID | Scenario | Expected | Result |
| --- | --- | --- | --- |
| SMART-004-QA-001 | Написать в группе фразу-задачу без команды | Бот корректно понимает/предлагает фиксацию без лишнего шума | Not run |
| SMART-004-QA-002 | Создать задачу короткой командой/ответом | Бот отвечает одной лаконичной строкой | Not run |
| SMART-004-QA-003 | Проверить задачу в приложении | Задача появилась у правильного пользователя/в правильном контексте | Not run |
| SMART-004-QA-004 | Нажать `✕` | Задача удаляется/отменяется, нет ghost task | Not run |
| SMART-004-QA-005 | Нажать `✏️` | Открывается ожидаемый edit/clarification path | Not run |
| SMART-004-QA-006 | Два участника пишут похожие задачи | Задачи не смешиваются между пользователями | Not run |
| SMART-004-QA-007 | Бот видит нерелевантное сообщение | Бот не спамит и не фиксирует мусор | Not run |
| SMART-004-QA-008 | Повторить одинаковую задачу быстро | Нет опасных дублей или они объяснимы | Not run |

## 3. Failure handling

| Failure | Severity | Action |
| --- | --- | --- |
| Задача попала не тому пользователю | P0/P1 privacy/data bug |
| Бот спамит группу | P1, fix before beta group use |
| Удаление `✕` не удаляет задачу | P1/P2 depending on visibility |
| Edit path не работает | P2 unless it blocks correction |
| Copy слишком длинная | P2/P3 polish depending on annoyance |

## 4. Done rule

`SMART-004` можно переводить в `Done`, только если:

- живой group smoke подтвердил создание задачи;
- задача видна в app у правильного пользователя;
- `✏️/✕` работают или имеют честный fallback;
- бот не отвечает на нерелевантный шум;
- P0/P1 findings заведены в `pm/bugs.md` или исправлены.
