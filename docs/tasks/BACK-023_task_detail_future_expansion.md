# BACK-023 — Расширение детальной карточки задачи после MVP

## Задача

Собрать будущие функции детальной карточки задачи в отдельный пакет, чтобы не перегружать первую волну `BACK-022`.

## Контекст

Алексей одобрил подход: сначала закрыть ручной MVP, затем расширять карточку только после проверки, что первая волна действительно помогает пользователю.

## Кандидаты на будущую волну

- Повторы задач: каждый день, неделю, месяц, свои дни.
- Вложения и ссылки: URL, фото, документы, сканы.
- Умные списки: сегодня, просрочено, ждёт ответа, важное, по человеку.
- Поиск и фильтры по тегам.
- Календарная синхронизация.
- Командные комментарии и ответственные.
- История изменений с понятными событиями: кто изменил срок, статус, описание.

## Условие старта

Не начинать до проверки `BACK-022` на реальном сценарии. Перед стартом нужен отдельный Decision Gate, если выбранные функции затрагивают хранение данных, privacy, командную работу, календарные интеграции или платные возможности.

## Критерий готовности будущего решения

Команда выбирает 1-2 функции из списка, оценивает влияние на UX, хранение данных, доверие и сроки, затем Алексей принимает отдельное решение.
