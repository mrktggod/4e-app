## Старт сессии

```bash
git fetch origin && git switch main && git pull --rebase
git switch -c feat/completed-tasks-week
```

## Что нужно сделать

В нижней части списка задач на дашборде (экран Home) добавить секцию с задачами, выполненными за последние 7 дней.

### 1. Секция «Выполнено за неделю»

- Показывать только задачи со статусом `done` и датой выполнения ≥ сегодня − 7 дней
- Располагать **ниже** всех активных задач
- Заголовок секции: `Выполнено за неделю (N)` — где N — количество задач
- Секция по умолчанию **свёрнута** (collapsed), разворачивается по тапу на заголовок
- Иконка стрелки на заголовке показывает состояние: ▶ свёрнуто / ▼ развёрнуто

### 2. Карточка выполненной задачи

- Использовать тот же компонент карточки что в BACK-019, но:
  - Текст названия с `opacity: 0.5` (приглушённый)
  - Зелёная галочка слева вместо номера с приоритетом
  - Без свайп-действий (задача уже завершена)
  - Дедлайн не показывать

### 3. Если выполненных задач нет

- Секцию не показывать совсем (не рендерить заголовок)

## Файлы для чтения

1. `FILE_MAP.md` → `FILE_MAP_UI.md` — найти функцию рендера списка задач (что-то типа `renderTasks` или `buildTaskList`)
2. Текущая карточка задачи из BACK-019 — найти CSS-класс `.task-card` и структуру
3. `AGENTS.md` — правила кодировки и git

## Критерий готовности

- Секция появляется ниже активных задач если есть done-задачи за 7 дней
- Секция свёрнута по умолчанию, разворачивается по тапу
- Счётчик N в заголовке актуален
- Выполненные карточки визуально отличаются (приглушённые, без свайпа)
- Если done-задач нет — секции нет
- Кириллица цела после правки (проверка по AGENTS.md Шаг 0)
