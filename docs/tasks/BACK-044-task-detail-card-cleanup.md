## Контекст

Алексей проверил детальную карточку задачи и отметил, что текущий порядок полей перегружает первый экран карточки. Нужно упростить визуальную структуру без добавления новой функции.

Эта задача только на UI-код. Реализация собрана в ветке `fix/task-detail-card-cleanup` (commit `250f35b`). По состоянию `pm/backlog.md` и `shared/ROADMAP.md` на 2026-07-21 задача закрыта как `Done`; текущий файл синхронизирован с каноническим PM-статусом без runtime-изменений.

## Что изменить

В экране `task-detail`:

1. Перенести текст описания/исходного запроса под заголовок задачи.
2. Убрать вкладку `Описание`, чтобы описание не дублировалось ниже.
3. Сделать `Срок` первым пунктом в блоке полей.
4. Быстрые кнопки `Сегодня / Завтра / Неделя` оставить сразу под `Срок`, но добавить заметный верхний отступ от строки срока.
5. Убрать из видимой карточки строки `Направление` и `Человек`.
6. Ниже оставить поля `Статус`, `Приоритет`, `Напоминание`.

Итоговый порядок:

```text
Заголовок задачи
Описание / исходный запрос

Срок
Сегодня / Завтра / Неделя
Статус
Приоритет
Напоминание

Чек-лист
Обсудить задачу / История
```

## Проверка

1. Открыть детальную карточку задачи на телефоне или mobile viewport.
2. Проверить, что описание видно под заголовком.
3. Проверить, что вкладки `Описание` больше нет.
4. Проверить порядок полей: `Срок → быстрые кнопки → Статус → Приоритет → Напоминание`.
5. Проверить, что строки `Направление` и `Человек` не видны в карточке.
6. Изменить срок, статус, приоритет и описание; убедиться, что сохранение не сломалось.
7. Проверить, что на mobile/web текст не налезает на поля и нижнюю навигацию.
