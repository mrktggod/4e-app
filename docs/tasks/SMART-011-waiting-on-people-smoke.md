# SMART-011 — waiting-on-people reminder smoke

Цель: проверить умный пинг `Жду от людей` в реальной группе/чате, не закрывая задачу по исходникам. Это социальная функция: важны не только API и кнопка, но и адресат, тон сообщения, отсутствие спама и idempotency.

## 1. Что уже реализовано

| Layer | Evidence |
| --- | --- |
| Reminder CTA | Кнопка `Напомнить N` доступна для задачи/ожидания |
| Bot action | Бот отправляет исполнителю вежливое напоминание |
| Product dependency | Требует корректной связки с `SMART-002` / исполнителями |

## 2. Live smoke checklist

| ID | Scenario | Expected | Result |
| --- | --- | --- | --- |
| SMART-011-QA-001 | В группе есть задача, назначенная не на Юрия | В UI/боте видно, кому можно напомнить | Not run |
| SMART-011-QA-002 | Нажать `Напомнить N` один раз | Сообщение уходит правильному исполнителю/в правильный чат | Not run |
| SMART-011-QA-003 | Проверить текст сообщения | Тон вежливый, без давления и без раскрытия лишней приватной информации | Not run |
| SMART-011-QA-004 | Повторно нажать быстро | Нет спама/дубликатов или есть понятный cooldown | Not run |
| SMART-011-QA-005 | Исполнитель без связанного Telegram | Показывается понятный fallback, не падает ошибка | Not run |
| SMART-011-QA-006 | Задача уже закрыта | Напоминание не отправляется или объясняется, что задача закрыта | Not run |
| SMART-011-QA-007 | Проверить историю/статус | Видно, что reminder был отправлен, если это предусмотрено UI | Not run |

## 3. Failure handling

| Failure | Severity | Action |
| --- | --- | --- |
| Reminder уходит не тому человеку | P0/P1 privacy/social bug, stop beta group rollout |
| Reminder спамится многократно | P1, add cooldown/idempotency |
| Bot падает или молчит без объяснения | P1/P2 depending on core impact |
| Copy звучит агрессивно | P2, fix before wider beta |
| Нет истории отправки | P3 unless users get confused |

## 4. Done rule

`SMART-011` можно переводить в `Done`, только если:

- live smoke прошёл в реальном Telegram/group context;
- адресат корректный;
- повторный клик не создаёт спам;
- copy принята Юрием/Алексеем как нормальная для реальных людей;
- P0/P1 findings заведены в `pm/bugs.md` или исправлены.
