# BACK-036 — варианты закрытия Telegram web fallback

## Контекст

Live smoke 2026-07-11 показал, что `https://t.me/Denzel89bot?start=auth_<token>` не отправляет `/start auth_<token>` автоматически, если у пользователя уже есть история чата с ботом. Telegram просто открывает существующий диалог. Поэтому текущая схема входа ненадёжна для старых пользователей бота.

Цель: выбрать решение, которое закрывает web-вход через Telegram для пользователей с уже существующей перепиской.

## Вариант A — ручная команда как fallback

Фронт после перехода на Telegram показывает пользователю точную команду:

```text
/start auth_<token>
```

На экране нужны:

- кнопка копирования команды;
- короткая подсказка: если бот сам не ответил, отправьте эту команду вручную в чат с ботом;
- сохранение текущей схемы возврата через `?telegram_start=<token>`.

Плюсы:

- минимальная реализация;
- сохраняет текущий worker/bot flow;
- можно быстро вывести в прод и закрыть блокер старых пользователей.

Минусы:

- ручной шаг снижает конверсию;
- пользователь может отправить команду не тому боту или закрыть экран;
- нужен аккуратный текст, чтобы сценарий не выглядел как ошибка.

Оценка: 0.5-1 день на frontend + smoke.

## Вариант B — Telegram Login Widget

Перейти на официальный Telegram Login Widget для web-входа. Он не зависит от истории чата с ботом и авторизует пользователя как сайтовый login-provider.

Плюсы:

- решает корневую проблему платформенного поведения `t.me/bot?start=...`;
- лучше UX для web-входа;
- меньше зависимости от ручных действий пользователя.

Минусы:

- потребуется отдельная backend-проверка подписи Telegram Login Widget;
- нужно аккуратно связать результат с существующей моделью аккаунтов и BACK-026;
- нужно отдельно проверить web, Telegram Mini App и существующий bot-side `/start auth_*` flow.

Оценка: 1.5-3 дня на worker + frontend + smoke.

## Рекомендация

Для быстрого закрытия регрессии взять вариант A как hotfix. Параллельно оставить вариант B как целевое решение для стабильного web-входа без зависимости от истории чата.

До выбора варианта `BACK-036` не переводить в `Done`.
