# 6. Ручной спринт — вечер 14 июля 2026 года

## Цель

За один вечер проверить только то, что невозможно надёжно закрыть автоматикой: реальные устройства, Telegram/VK, РФ-сети, человеческое понимание Home и разрешённые платёжные flows.

Ожидаемый результат — таблица `Pass / P0 / P1 / P2`, а не попытка исправлять всё по ходу.

## До начала — готовит команда

- [ ] Один точный staging URL.
- [ ] App commit SHA.
- [ ] Worker commit/deployment version.
- [ ] Два dev/test аккаунта с Premium и seed data.
- [ ] Один fresh account для onboarding/referral.
- [ ] TG bot username и VK production/staging entry без поиска по чатам.
- [ ] Две независимые РФ-сети/оператора.
- [ ] Список разрешённых реальных/test платежей.
- [ ] Место для скриншотов/видео и результатов.
- [ ] Никаких паролей, токенов, confirm codes или платёжных данных в этом Markdown.

Если candidate меняется после начала, спринт перезапускается только для затронутой зоны; старый результат помечается SHA, а не стирается.

## Блок 3 — первый результат и Home, 25 минут

### Fresh user

- [ ] Понимает, что делает 4, без объяснения голосом разработчика.
- [ ] Создаёт задачу текстом.
- [ ] Создаёт задачу голосом.
- [ ] Видит реальный персональный план, не демо-данные.
- [ ] Понимает первый следующий шаг.
- [ ] Home не показывает отрицательное время после 22:00.
- [ ] Tap/long-press центральной кнопки не конфликтуют.
- [ ] Три priority cards открывают правильные задачи.
- [ ] Фильтры задачи/обещания/выполнено ведут в один рабочий список.

Если `ONBOARD-001` ещё не реализован, измерить текущий time-to-first-plan вручную и записать, где понадобилась подсказка. Это baseline, а не Pass.

## Блок 4 — mobile UX, voice и accessibility, 20 минут

- [ ] Telegram WebView запускает voice.
- [ ] После распознавания задача/ответ появляются без непонятной задержки.
- [ ] При запрете микрофона есть понятный текстовый fallback.
- [ ] AI input не перекрыт клавиатурой.
- [ ] Контент не уходит под bottom nav/safe area.
- [ ] Последнее действие профиля доступно.
- [ ] Keyboard/focus проходит auth forms.
- [ ] Dialog/bottom sheet возвращает фокус и закрывается понятно.

Для voice записать один `window.__voicePerfLast` sample или видео с таймингом, не только ощущение «медленно».

## Блок 5 — выбранные платежи, 30–45 минут

### Stop condition

Не проводить реальную оплату, пока backend выбранного канала не проверяет подлинность, сумму, план и повтор транзакции. UI success недостаточен.

### CloudPayments/card

- [ ] Правильный plan/amount отображаются.
- [ ] Provider success сопоставлен с backend transaction.
- [ ] Неправильная подпись/сумма отвергается автоматическим негативным тестом до ручного flow.
- [ ] Повтор callback не продлевает Premium второй раз.
- [ ] `/auth/me`/subscription показывает paid period.

### Telegram Stars

- [ ] Invoice создан для правильного plan/amount.
- [ ] Пользователь подтверждает оплату внутри Telegram.
- [ ] `successful_payment` обрабатывается доверенным bot path.
- [ ] Публичный client request не может сам активировать Premium.
- [ ] Повтор update идемпотентен.

### VK Pay

- [ ] OrderBox открывается внутри реального VK Mini App.
- [ ] UI success подтверждён backend/provider verification.
- [ ] Канонический пользователь становится paid на сервере.
- [ ] Повторное открытие приложения сохраняет Premium.

Достаточно пройти один канал полностью. Другие получают статус и отдельный следующий шаг.

## Блок 6 — referral, 15–20 минут

Только если candidate содержит referral и команда понимает текущий reward rule.

1. User A копирует ссылку.
2. Fresh User B открывает её на поддерживаемой поверхности.
3. B регистрируется.
4. B создаёт первую задачу/план.
5. Проверяется момент reward согласно выбранному правилу.
6. A получает один bonus и `referralCount +1`.
7. Повтор/самореферал не начисляет бонус.
8. В аналитике видны open/signup/activation/reward.

Если VK flow отсутствует, не считать это общеплатформенной рефералкой. Записать `TG/web only`.

## Блок 7 — короткий смысловой тест, 10 минут

Попросить человека, который не разрабатывал экран, ответить:

1. Что это за продукт?
2. Что он предлагает сделать сейчас?
3. За что здесь платить?
4. Что произойдёт после окончания trial?
5. Что вызывает недоверие?

Записать слова дословно. Не убеждать участника во время ответа.

## Правило после спринта

1. P0 — исправлять первым.
2. P1 — только если блокирует beta/activation/payment.
3. P2 — записать и не расширять текущий slice.
4. Pass — обновить backlog evidence и больше не перепроверять без изменения кода/окружения.
5. Не переводить `Done`, если проверен staging, а production не содержит изменение.
