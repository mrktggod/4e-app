# INFRA-006 — единая рабочая копия Project 4

Дата подготовки: 2026-07-16.

Цель: убрать риск, когда работа делается в одной локальной копии, а проверка/деплой/коммит идут из другой.

Важно: этот документ ничего не удаляет. Удаление или архивирование копий — только после ручного подтверждения Юрия/Алексея.

## Известная проблема

В проекте уже были инциденты:

- правки попадали в неактуальную локальную копию;
- часть ночной работы пришлось восстанавливать через cherry-pick/ручную сверку;
- рядом существуют разные поверхности: `.tmp-4e-app-publish`, `4e-worker`, `4e-worker-p0`, `Documents/4/4e-app`, desktop/session copies.

## Предлагаемая каноничная схема

Каноничные рабочие папки:

- App/frontend: `X:\4\.tmp-4e-app-publish`
- Worker/API: `X:\4\4e-worker`

Все остальные копии должны стать одним из трёх типов:

- `archive`: только исторический read-only backup;
- `scratch`: временная песочница без commit/push;
- `delete later`: удалить после ручной сверки, что там нет уникальной работы.

## Правило перед любой новой задачей

Перед правками:

```powershell
cd X:\4\.tmp-4e-app-publish
git status --short
git rev-parse --show-toplevel
git branch --show-current
git rev-parse HEAD
git rev-parse origin/feat/admin-tariff-api
```

Для worker:

```powershell
cd X:\4\4e-worker
git status --short
git rev-parse --show-toplevel
git branch --show-current
git rev-parse HEAD
git rev-parse origin/main
```

Если `HEAD != origin/...`, сначала понять почему, а не начинать новую задачу.

## Маркировка копий

В каждой неканоничной копии добавить локальный файл без commit:

```text
DO_NOT_WORK_HERE.txt
```

Содержимое:

```text
Эта папка не является каноничной рабочей копией Project 4.
Перед любыми правками перейди в:
- App: X:\4\.tmp-4e-app-publish
- Worker: X:\4\4e-worker
```

Если копия всё ещё нужна как архив, лучше переименовать папку с префиксом:

```text
_archive_DO_NOT_WORK_HERE_...
```

## Что нельзя делать автоматически

- Нельзя удалять папки без ручной сверки.
- Нельзя делать `git reset --hard` в копии с неизвестными изменениями.
- Нельзя переносить патчи между копиями без просмотра diff.
- Нельзя считать `git stash` надёжной аварийной копией при риске mojibake/encoding.

## Безопасный аудит копии

Для каждой найденной копии:

```powershell
git status --short
git remote -v
git branch --show-current
git log -1 --oneline
```

Классификация:

- чистая и совпадает с origin: можно архивировать/удалить после подтверждения;
- есть untracked/modified: сохранить diff/копии файлов перед решением;
- нет `.git`: считать обычной папкой, сравнивать вручную по файлам, не принимать как source of truth.

## Критерий Done

`INFRA-006` можно закрывать, когда:

- Юрий/Алексей явно выбрали каноничные app/worker папки;
- все неканоничные копии помечены как archive/scratch/delete later;
- в неканоничных копиях нет незакоммиченной уникальной работы;
- новые задачи в Codex стартуют только из каноничной папки;
- это правило зафиксировано в `AGENTS.md` или отдельном process note.
