# Assistant Evaluation

## Рубрика качества

Оценивай ответы ассистента по шкале 1-5.

| Критерий | Что значит 5/5 | Оценка | Комментарий |
| --- | --- | --- | --- |
| Полезность | Ответ сразу помогает сделать следующий шаг |  | |
| Точность | Нет выдуманных фактов, дат, статусов, ссылок |  | |
| Контекст проекта | Учитывает текущие цели, roadmap и ограничения |  | |
| Структура | Информация легко превращается в задачу или решение |  | |
| Проактивность | Предлагает разумный следующий шаг без хаоса |  | |
| Тон | Звучит как проектный помощник, а не случайный чат |  | |

## Набор стартовых проверок

- [ ] Сырой список багов превратить в таблицу с приоритетами.
- [ ] Из заметок встречи собрать решения и action items.
- [ ] Из roadmap выделить риски и зависимости.
- [ ] Подготовить короткий статус проекта для руководителя.
- [ ] Составить smoke checklist для новой функции.
- [ ] Найти противоречия в требованиях.
- [ ] Разделить входящий поток на bugs, features, questions и tech debt.

## Эталон SMART-009 / SMART-014 — несколько дел голосом

**Назначение:** проверить, что 4 отделяет самостоятельные задачи, предлагает решение без выдуманных данных и задаёт вопрос только при действительно важной неопределённости.

### Вход

```text
Сегодня нужно позвонить поставщику, закончить отчёт,
записаться к врачу, забрать заказ и не забыть написать Юре.
```

### Ожидаемое поведение

1. Распознаны пять отдельных задач.
2. Сроки не добавлены.
3. `Записаться к врачу` не превращено в поездку.
4. Поставщик и Юра показаны рядом, но сохраняются отдельно.
5. AI предлагает один фокус и прямо помечает основание как предположение.
6. Вопрос про врача, длительность или отсутствующие сроки не задаётся.
7. До подтверждения ничего не сохраняется.
8. CTA точно описывают результат.

### Эталонный ответ

```text
Я услышал 5 дел. Сроки не ставлю — ты их не называл.

Предлагаю такой порядок:

Главный фокус
• Закончить отчёт.

Короткий коммуникационный блок
• Позвонить поставщику.
• Написать Юре.
Сохраню как две отдельные задачи.

Остальные дела
• Записаться к врачу.
• Забрать заказ.

Отчёт ставлю первым как наиболее объёмное дело. Это моя оценка,
а не факт из твоей фразы.

[Добавить 5 задач] [Выбрать другой фокус] [Добавить без приоритета]
```

### Критические ошибки

- выдуман дедлайн или длительность;
- появилась поездка к врачу или свободное окно календаря;
- несколько дел сохранены одной задачей;
- связанные дела автоматически слиты;
- задачи записаны до подтверждения;
- задан вопрос, который не влияет на корректность данных;
- кнопка называется `Да` или `Ответить` и не объясняет последствие.

### Решение

- `Pass`: все восемь требований выполнены, критических ошибок нет.
- `Needs Work`: задачи выделены верно, но есть лишний вопрос, слабая формулировка фокуса или неясный CTA.
- `Fail`: выдуманы данные, потеряны/слиты задачи или произошла запись без подтверждения.
