# BETA-001 — закрытый тест 5-10 пользователей

Дата подготовки: 2026-07-16.

Цель: запустить короткий закрытый тест без раздувания скоупа и быстро понять, где продукт ломается перед расширением доступа.

## Кого звать

- 5-10 человек, которые реально ведут задачи/обещания каждый день.
- Минимум 2 пользователя Telegram, минимум 2 пользователя web/PWA.
- Если есть возможность: 1 пользователь VK Mini App.
- Не брать людей, которым нужно долго объяснять саму идею продукта: на этом этапе проверяем не презентацию, а рабочий daily-flow.

## Что дать тестеру

Короткое сообщение:

```text
Привет! Даю закрытый доступ к 4 — это личный AI-штаб дня.

Пожалуйста, проверь 3 сценария:
1. Создай 3-5 задач на сегодня/неделю.
2. Попроси 4 выбрать главный фокус дня.
3. Закрой хотя бы одну задачу и попробуй поделиться планом/итогами.

Если что-то сломается — пришли скрин и коротко: где нажал, что ожидал, что реально произошло.
```

## Обязательный smoke перед приглашением

- Email signup/login на staging или production.
- Telegram fallback login, если тестер идёт через Telegram.
- Home открывается после login.
- Создание задачи работает.
- AI-chat не падает на первом сообщении.
- Paywall не блокирует бесплатный onboarding/trial путь.

## Сценарий дня 1

1. Тестер регистрируется.
2. Добавляет 3 задачи.
3. Открывает home и смотрит top priorities.
4. Пробует голосовой ввод или AI-chat.
5. Закрывает одну задачу.
6. Нажимает `Поделиться планом дня` или `Итоги недели`.

Успех дня 1:

- Пользователь сам дошёл до home.
- Создал хотя бы одну реальную задачу.
- Понял, что делать дальше, без объяснения по Zoom/чату.

## Сценарий дня 2-3

1. Вернуться в приложение без повторного объяснения.
2. Проверить, сохранились ли задачи.
3. Добавить новую задачу.
4. Попробовать обещание/исполнителя, если сценарий естественный.
5. Отправить один feedback.

Успех дня 2-3:

- Пользователь вернулся сам.
- Есть хотя бы один повторный полезный action: новая задача, done, AI-chat, share, reminder.

## Что фиксировать в feedback

Формат одной записи:

```text
Кто:
Поверхность: web / PWA / Telegram / VK
Сценарий:
Где нажал:
Что ожидал:
Что реально увидел:
Скрин/видео:
Влияние: blocker / painful / cosmetic / idea
```

## Критерии выхода из закрытого теста

Можно расширять доступ, если:

- Нет P0/P1 blockers в auth/signup/login.
- Нет потери задач после reload/reopen.
- Минимум 3 тестера создали реальные задачи.
- Минимум 2 тестера вернулись на второй день.
- Есть понятный список top blockers в `pm/bugs.md`.

Остановить расширение, если:

- Auth ломается у 2+ тестеров.
- Задачи не сохраняются или пропадают.
- AI-chat массово ведёт в тупик.
- Paywall/entitlement случайно блокирует trial/free flow.

## Что обновить после теста

- `pm/bugs.md`: все реальные баги.
- `pm/backlog.md`: статусы BETA/FEEDBACK и новые P0/P1.
- `shared/WORK_LOG.md`: короткий итог beta-run.
- `pm/team-sync.md`: что проверено руками и кто владелец следующего шага.
