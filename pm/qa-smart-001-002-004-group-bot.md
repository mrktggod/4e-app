# QA — SMART-001 / SMART-002 / SMART-004

Цель: вручную проверить группового Telegram-бота на staging после обновления runtime.

## Что проверить

1. Однострочное подтверждение задачи
   - В тестовой группе написать обычную задачу без `@username`.
   - Ожидание: бот отвечает одной строкой в формате `✓ Имя: задача — срок`.
   - Ожидание: под ответом есть только кнопки `✏️` и `✕`.

2. Исполнитель через `@username`
   - Написать `@username` + текст задачи.
   - Ожидание: в сохранённой задаче есть `assigneeTgId` и `assigneeUsername`.
   - Ожидание: имя исполнителя в ответе совпадает с упомянутым участником.

3. Исполнитель через reply
   - Ответить на сообщение участника и в reply написать задачу.
   - Ожидание: бот назначает задачу автору исходного сообщения.

4. Ростер группы: join / leave
   - Добавить участника в тестовую группу.
   - Создать задачу на него по `@username` или по имени.
   - Удалить участника из группы.
   - Ожидание: join попадает в roster, leave не ломает разбор и не оставляет битый сценарий.

## Кнопки

- `✏️` открывает сценарий редактирования без дубля старой задачи.
- `✕` удаляет задачу и не показывает ложное сообщение «Отменено», если контекст уже потерян.

## Дополнительно для INFRA-002

- Открыть `https://app.4-ai.site/` без VPN со своего оператора.
- Зафиксировать результат отдельно от группового smoke: это не блокер для staging-бота, но важная точка в расследовании масштаба РФ-ограничений.
