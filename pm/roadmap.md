# Roadmap — 4 AI-секретарь

## Vision

Персональный операционный директор — партнёр, который следит за всем, напоминает, анализирует и берёт рутину на себя.

## North Star

Пользователь открывает 4 утром и за 30 секунд видит что важно сегодня, что горит и что ждёт команда.

## Продуктовые столпы

| # | Столп | Суть |
| --- | --- | --- |
| 1 | AI-советник | Лёгкий доступ к анализу и советам в любой момент |
| 2 | Дашборд жизни | Всё важное в одном месте, собирается само |

### Регистрация и авторизация — детализация

Цель: пользователь должен безопасно и без тупиков пройти весь путь от первого открытия до подтверждённого аккаунта, восстановления доступа и выхода из профиля.

| Блок | Что нужно закончить | Приоритет | Готово когда |
| --- | --- | --- | --- |
| UX-сценарии | Описать полный flow: onboarding → регистрация → вход → Telegram/VK вход → восстановление пароля → reset → logout → повторный вход | P0 | Есть карта экранов, переходов, ошибок и пустых состояний |
| UI состояния | Доработать меню входа/регистрации: tabs, loading, disabled-кнопки, inline-ошибки, success-состояния, retry, fallback при таймауте | P0 | Нельзя получить пустой экран или непонятный статус после любого auth-действия |

#### Definition of Done

- [ ] Новый пользователь может зарегистрироваться без тупиков и непонятных ошибок.
- [ ] Существующий пользователь может войти через email/password.
- [ ] Telegram/VK вход работает или показывает понятный fallback.
- [ ] Пользователь может восстановить доступ через forgot/reset password.
- [ ] Logout очищает локальное состояние и возвращает на login.
- [ ] Приватные экраны без валидного токена не открываются.
- [ ] Старые, битые и истёкшие токены не дают доступ к данным.
- [ ] Нет пустых экранов после login, register, reset, logout, VK/TG auth и таймаутов.
- [ ] Ошибки написаны понятно для пользователя и не раскрывают лишнюю security-информацию.
- [ ] Rate limit/throttling включён на чувствительных auth-действиях.
- [ ] Пароли, reset-token и session-token не логируются и не хранятся в открытом виде.
- [ ] QA regression пройден на mobile, VK Mini App, Telegram Mini App и обычном браузере.
- [ ] 152-ФЗ, privacy-ссылка и пользовательские согласия учтены в регистрационном flow.

#### Security acceptance checklist

- [ ] Пароли передаются только по HTTPS/TLS.
- [ ] Пароли хранятся только как стойкий хэш с солью, без plaintext и обратимого шифрования.
- [ ] Минимальная длина пароля и проверка слабых/частых паролей; без устаревших правил вида "обязательно спецсимвол".
- [ ] Ошибки регистрации, входа и восстановления не раскрывают, существует ли конкретный email.
- [ ] Есть rate limit/throttling на login, register, forgot-password и reset-password.
- [ ] Reset-token одноразовый, короткоживущий, хранится безопасно и инвалидируется после использования.
- [ ] После сброса пароля выполняется повторная авторизация или инвалидируются старые сессии.
- [ ] Logout очищает локальные токены и состояние пользователя.
- [ ] Все приватные экраны закрыты auth-guard'ом.
- [ ] Чувствительные действия требуют повторной проверки пользователя.
- [ ] Логи не содержат пароли, reset-token, session-token, access-token целиком.
- [ ] Добавлены базовые security headers и проверены XSS/CSRF риски для auth-flow.
- [ ] Подтверждена совместимость с требованиями 152-ФЗ: privacy, согласия, хранение и обработка персональных данных.

Ориентир по лучшим практикам: OWASP Authentication Cheat Sheet и OWASP Session Management Cheat Sheet.
