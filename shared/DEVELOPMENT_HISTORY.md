## Архитектурные решения (принятые и необратимые)

| Решение | Почему |
|---------|--------|
| Telegram Mini App + VK Mini App — отдельные файлы | VK не поддерживает Telegram SDK, шрифты Google, внешние ресурсы |
| Cloudflare Worker (минифицирован) | Edge-вычисления, нет холодного старта |
| GitHub Pages для фронтенда | Бесплатно, автодеплой через push |
| Railway для бота | Автодеплой, webhook polling |
| `chatId` передаётся явно, не через глобал | Иначе KV ключи путались между telegramId и chatId |
| HMAC-подпись запросов бот→воркер | Безопасность: закрывает уязвимость открытого API |

## Паттерны ошибок (учить наизусть)

1. **Кодировка** — самая частая. PowerShell без явного UTF-8 → кириллица становится `????????`. Всегда проверять `Select-String -Pattern 'Войти|Задачи'`.
2. **KV + chatId** — путаница между `telegramId` и `chatId`. Правило: chatId передавать явно.
3. **Revert циклы** — не тестировали перед коммитом, потом fix → revert → fix снова.
4. **VK iframe ограничения** — Google Fonts, Telegram SDK, большинство внешних ресурсов недоступны.
5. **Codex работает параллельно** — конфликты при pull. Всегда `git pull --rebase` перед пушем.
6. **Секреты в коде** — ANTHROPIC_KEY закоммичен был 2026-06-24. Правило: только PLACEHOLDER в worker.js.
